# BakeMeter 🌡（化けメーター）

**Linux LLMサーバーの Thunderbolt eGPU 凍結を予知する早期警報システム。**

🇬🇧 English version → [README.md](README.md)

Thunderbolt接続のeGPUでLLM(Ollamaなど)を動かしていて、長い生成の途中でマシンが
突然固まる — その症状、私たちが測ったのと同じ病気かもしれません。TBリンク上で
訂正可能なPCIe `BadDLLP` エラーが絶え間なく流れ続け、ごく稀に致死性のエラーが
混ざってバスごと死にます。電源のせいでも熱のせいでもなく、ケーブル・アダプタ・
筐体・ホストを交換してもエラー率は変わりませんでした。実測データは
[docs/findings.md](docs/findings.md) をどうぞ。

朗報: **凍結は予知できます。** 致死エラーの前には、訂正可能エラーの増加が
何時間も先行します。BakeMeterはそれを数え、クラッシュの前にGPUの負荷を
降ろします。

## クイックスタート

```bash
# 1. エラー水位の見張り(cronで5分ごと)
*/5 * * * * bash /path/to/scripts/bake_meter.sh

# 2. 自分の環境のエラー率を再現・計測(ほぼ安全・スクリプト冒頭の注意を参照)
MODEL=llama3:8b bash scripts/burst_test.sh
```

`bake_meter.sh`(v2.1)は **sysfsの実AERカウンタ** を読み、5分間の増分をCSVに
記録します。**1,000件/5分で注意(WARN)**、**5,000件/5分で危険(DANGER)** と判定し、
危険への遷移時に一度だけ全Ollamaモデルを退避(その後、退避が本当に成功したかを
再確認)します。転送が止まり、凍結を回避できます。モデルは次の利用時に自動で
再ロードされます。危険の後は **COOLDOWN(冷却)水位を30分** — かつ直近サンプルが
静かになるまで — 維持します。危険状態は転送停止より長生きし、リンクが静かに
なった約30分後に致死エラーが来た実測があるためです(findings #6)。
sysfs AERが読めない環境では `journalctl` 数え(50件/hで注意・200件/hで危険)へ
フォールバックしますが、journalctlはレート制限により**実際の約34分の1しか
数えられない**ことに注意してください(findings #8)。フォールバック中も sysfs の
基準値は上書きしないため、デバイスが消えて戻ってきても(GPU脱落がまさにこれ)
偽のバーストには化けません。しきい値やパスはすべて `BM_*` 環境変数で変更できます
(スクリプト冒頭参照)。下流アプリの安全ゲート用に `bake_state.json` を毎回
アトミックに書き出し、`test_bake_meter.sh`(偽sysfsで実スクリプトを回す
13ケースの回帰テスト・root/GPU不要)を同梱しています。

## 凍結ほぼゼロ運用の作法(実測から)

1. 主力モデルは常駐させる(`keep_alive -1`) — モデル載せ替えは約3〜5エラー/GBの代償。
2. 生成の長さに上限を — トークン出力は約1〜3エラー/秒の代償。
3. BakeMeterを安全網として回し続ける。
4. 根治はネイティブPCIeスロットだけ。それまでの延命はこれで。

## Mac Pro 2013(ゴミ箱)の同志へ

あなたのマシンは2026年でも現役のLLMサーバーになれます — うちはRTX 3090を
TB2で毎日回しています。苦労して得たコツ: eGPUは**下段のTBバスでしか起動しない**、
**電源投入前に接続必須**(ホットプラグは絶対に認識されない)、そして長い生成中の
凍結は上記のリンクエラー病 — 測れて、予知できて、BakeMeterで生き延びられます。
ゴミ箱を捨てないで。🗑✨

## 使った結果を教えてください 📬

動いた・動かなかった、どちらの報告も歓迎です — 1台ぶんの実測が丸ごと貴重な
データポイントになります。[Issue](../../issues) にだいたいこの形で書いて
もらえると助かります(日本語でOK):

```
ホスト / TB世代    : (例: Mac Pro 2013, TB2 / ThinkPad X1, TB4)
筐体とGPU          : (例: Razer Core X + RTX 3090)
OS / カーネル      : (例: Ubuntu 26.04, 6.14)
BadDLLP 平常時     : (アイドル時の5分あたりエラー数)
負荷時             : (生成中・モデル載せ替え中の5分あたりエラー数)
凍結を回避できた?  : はい / いいえ / そもそも凍結しない
備考               : 気づいたことなんでも
```

うちの最新実測(2026-07-12): **17〜18GBモデルのロード/アンロード8回で、25分間に
BadDLLP +22.8万件** — モデルの載せ替えが、私たちが測った中で断トツ最大の
引き金でした。主力モデルは常駐させましょう。

## 現在地 / 貢献について

2026-07-10、Mac Pro 2013(TB2) + Razer Core X + RTX 3090 という「世界で一番
無理のあるLLMサーバー」での一晩の統制実験から生まれました — 無理があるから
こそ、これが必要だったのです。他のTBホスト(TB3/TB4/USB4)でのPR・Issue・実測
報告を心から歓迎します。`burst_test.sh` の数字とマシン構成を添えてどうぞ。
